Getting Started.

You can deploy this static website on AWS using ElasticBox and Docker. We have created a very simple Docker container in ElasticBox that allows you to take static html from a Github repo and deploy on your Amazon Web Services account or other provider of your choice.

In order to deploy this website, you’ll need a few things. First, you will need an account on ElasticBox. If you don’t already have an account, you can sign up here. You’ll also need an account on Amazon Web Services. If you don’t already have an account, you can setup your account here.

Once that’s done, you can deploy this site within minutes by following the steps below!

What is ElasticBox and what are Boxes?

ElasticBox makes it easy for developers and enterprises to deploy and manage their applications on any cloud infrastructure. ElasticBox provides a simplified, modular, service based application development structure, where fully configured components of an application architecture are encapsulated as “Boxes” and made available as a service.

Boxes are reusable, versionable, available as a service, and then fully mobile across cloud environments. To create a multi-tier application architecture, one simply “stacks” Boxes. The result? DevOps at your fingertips.

What is Docker and what are Containers?

Docker is an open-source project to easily create lightweight, portable, self-sufficient containers from any application. The same container that a developer builds and tests on a laptop can run at scale, in production, on VMs, bare metal, OpenStack clusters, public clouds and more.
