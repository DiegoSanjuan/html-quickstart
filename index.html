Getting Started.

Want to use Docker containers in your deployments? This quick start gives a feel for launching a docker environment quickly using ElasticBox. Here, you have a pre-built Docker box that lets you deploy a static web page to AWS in an isolated container based on Docker.

By the end of the quick start, you should be able to start using the sample box as a template to build your own Docker deployments.

Before you begin, you’ll need a few things.

  • First, you need an account on ElasticBox. If you don’t have one, sign up here.
  • And an account on Amazon Web Services.

Deploy This Website With Docker And ElasticBox.

In order to deploy this web page on AWS, follow the simple instructions below:

  1. On the ElasticBox Providers page, add your AWS account with an access key ID and secret. If you don't have your account key and secret, visit this link. ElasticBox brings up all the AMIs in your account.
  2. If you don’t have already have an AMI based on Ubuntu 13.04, find it using the Ubuntu locator. Then add the AMI to your AWS account under Machine Images. You need this AMI because docker only works with this Ubuntu Machine Image on AWS.
  3. You’re now ready to create an instance of Docker to serve a simple web page.
    • From the Instances page, click New Instance.
    • Find and select the Docker box. You get a New Instance dialog.
    • In the dialog, give an environment name like dev, test, or staging to identify the deployment.
    • Create a new deployment profile to verify deployment settings. Select your AWS account and make sure you have the right Ubuntu AMI image selected. Add a key pair to be able to SSH into the instance once launched.
    • Save the deployment profile and click Deploy.

Summary: When you deploy, the box creates an Ubuntu Docker container based on the PHP, Apache that’s accessible through port 80. In the container, the box uses a GitHUB box to install Git using YUM (a package management tool), and uses the Git repository box to clone a sample web page to the Apache directory. Through the port endpoint, you can now access the web page served on Apache in the Docker environment.

What is ElasticBox and what are Boxes?

ElasticBox makes it easy for developers and enterprises to deploy and manage their applications on any cloud infrastructure. ElasticBox provides a simplified, modular, service based application development structure, where fully configured components of an application architecture are encapsulated as “Boxes” and made available as a service.

Boxes are reusable, versionable, available as a service, and then fully mobile across cloud environments. To create a multi-tier application architecture, one simply “stacks” Boxes. The result? DevOps at your fingertips.

What is Docker and what are Containers?

Docker is an open-source project to easily create lightweight, portable, self-sufficient containers from any application. The same container that a developer builds and tests on a laptop can run at scale, in production, on VMs, bare metal, OpenStack clusters, public clouds and more.
